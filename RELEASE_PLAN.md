# GWS-Auto-for-Android: Google Playストア リリース計画書

このドキュメントは、GWS-Auto-for-AndroidアプリケーションをGoogle Playストアで新規リリースするために必要な手順をまとめたものです。

---

### フェーズ1: アプリケーションの最終準備（コードとアセット）

このフェーズでは、アプリケーションのコードベースとアセットを本番リリースに向けて準備します。

- [ ] **アプリケーションIDの確定:** `app/build.gradle.kts`内の`applicationId`を最終的なものに確定します。**これは最初のリリース後に二度と変更できません。**
- [ ] **バージョン情報の設定:** `app/build.gradle.kts`で`versionCode`を更新し、ユーザーに表示される`versionName`（例: "1.0.0"）を設定します。
- [ ] **デバッグ用コードの削除:** `Log.d`のようなデバッグ用ログ、テストコード、不要なコメントアウトが残っていないかを確認・削除します。
- [ ] **APIキーの保護:** 機密性の高いキーがコードに直接書き込まれていないことを確認します。（現在のGitHub Secretsを利用する方法は適切です）。
- [ ] **最終版アプリアイコンの作成:** アダプティブアイコン（フォアグラウンドとバックグラウンドのレイヤー）を高解像度で準備します。
- [ ] **オープンソースライセンス画面の実装（推奨）:** アプリ内で使用しているサードパーティライブラリのライセンス情報を表示する画面を作成します。

---

### フェーズ2: 法務およびコンプライアンス対応

このフェーズでは、アプリが法的な要件とGoogle Playのポリシー、特にユーザーデータのプライバシー保護に準拠していることを確認します。

- [ ] **プライバシーポリシーの作成:**
    - [ ] 収集するデータの種類（Firebaseが収集するAnalytics, Auth, Crashlyticsのデータを含む）、その利用目的、共有先を明記したプライバシーポリシーを作成します。
    - [ ] プライバシーに関する問い合わせ先を記載します。
    - [ ] プライバシーポリシーを公開し、誰でもアクセスできるURLを準備します。
- [ ] **アセットの権利確認:** アプリ内で使用するすべての画像、アイコン、フォント等の素材が、商用利用可能なライセンスに基づいていることを確認します。

---

### フェーズ3: Google Play Consoleの設定

このフェーズでは、Google Play Console上でアプリの公開情報を設定します。

- [ ] **デベロッパーアカウントの作成:** 未作成の場合、Google Playデベロッパーアカウントを登録します。
- [ ] **新しいアプリの作成:** Play Consoleで新しいアプリケーション情報を登録します。
- [ ] **ストア掲載情報の設定:**
    - [ ] 最終的なアプリ名、短い説明、詳細な説明を記述します。
    - [ ] 高解像度のプロモーション用アセットをアップロードします:
        - [ ] アプリアイコン (512x512)
        - [ ] フィーチャーグラフィック (1024x500)
        - [ ] スマートフォン用のスクリーンショット (最低2枚)
        - [ ] タブレット用のスクリーンショット (7インチと10インチ、任意だが推奨)
- [ ] **コンテンツのレーティング設定:** アプリの対象年齢を決定するためのアンケートに回答します。
- [ ] **データセーフティセクションの入力:**
    - [ ] プライバシーポリシーに基づき、アプリのデータ収集・共有について正確に申告します。Firebase等のサードパーティSDKが収集するデータも対象です。
    - [ ] 公開したプライバシーポリシーのURLを登録します。
- [ ] **価格と配信国の設定:** アプリを無料にするか有料にするか、どの国で配信するかを設定します。

---

### フェーズ4: リリースビルドの作成と公開

この最終フェーズでは、署名済みのリリースビルドを作成し、ユーザーに公開するまでの技術的な手順をカバーします。

- [ ] **リリースキー（キーストア）の生成と保管:**
    - [ ] 本番リリース用のキーストアファイル（`.jks`または`.keystore`）を生成します。
    - [ ] **【最重要】** キーストアファイルとそのパスワードは、複数の安全な場所にバックアップしてください。**これを紛失すると、二度とアプリを更新できなくなります。**
- [ ] **Gradleでの署名設定:** `app/build.gradle.kts`の`signingConfigs`ブロックで、リリースキーを使用する設定を行います。（現在は環境変数/Secretsから読み込む設定ができています）。
- [ ] **Android App Bundle (AAB) のビルド:**
    - [ ] `./gradlew bundleRelease` コマンドを実行し、署名済みのAABファイルを生成します。
- [ ] **内部テストトラックへのアップロード:**
    - [ ] Play Consoleの「内部テスト」トラックに、生成したAABファイルをアップロードします。
    - [ ] テスターのメールアドレスをリストに追加します。
- [ ] **最終テストの実施:** テスターはPlayストアから提供されたリンク経由でアプリをインストールし、最終確認を行います。
- [ ] **製品版へのプロモート:**
    - [ ] テストで問題がなければ、「内部テスト」から「製品版」トラックへビルドをプロモート（昇格）させます。
    - [ ] リリースを申請します。Googleによる審査（数時間〜数日）が完了すると、ストアで公開されます。
