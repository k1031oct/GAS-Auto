プロジェクト AI エージェント ルールブック

これは、このリポジトリのコード修正およびレビューを行う AI エージェント（@gemini-cli）のための基本ルールとコンテキストです。

1. 開発の目的と技術スタック

主な目的: このプロジェクトはウェブアプリケーション開発であり、ユーザー体験とセキュリティを最優先とします。

使用技術: [例: JavaScript (ES6+), Tailwind CSS, Node.js v18]

コーディング規約: [例: ESLint/Prettierのルールに従う。変数名はキャメルケースを使用。]

2. Issue 処理のルール（自律実行の基準）

AIエージェントは、以下の条件を満たした Issue のみを自動で処理し、Pull Request を作成します。

必須トリガー

Issueのラベルに ready-for-ai が付与されていること。

必須の記述要素

Issue本文には、以下の項目が明確に含まれていること。

ゴール: 達成すべき具体的な結果 (例: ログイン機能を修正)。

ターゲットファイル: 修正対象のファイルパスと関数名 (例: src/auth.jsのloginUser関数)。

テスト基準: 修正後にパスすべきテストの条件（例: 既存のユニットテストをすべてパスすること、または特定の新しいテストコードが追加されていること）。

3. コミットメッセージのフォーマット

コミットメッセージは必ず英語で記述し、動詞から始めること (例: Fix: Handle null pointer exception in auth module)。

コミットは、Issueの番号（例: #123）で参照すること。

4. 実行後の行動

コード修正後は、必ず新しい Pull Request を作成し、人間による最終レビューを要求すること。

## ステップ 3: GitHub Actions ワークフローの自動生成

Gemini CLIの組み込みコマンドを使用し、自律的なエージェントワークフローをリポジトリに自動で追加します。

### 3.1 ワークフローファイルの生成

1.  ローカルリポジトリのルートディレクトリで **Gemini CLI** を起動します。
    ```bash
    gemini
    ```
2.  Gemini CLI内で、以下のセットアップコマンドを実行します。
    ```bash
    /setup-github
    ```
    このコマンドにより、`.github/workflows` ディレクトリ内に必要なYAMLファイルが自動的に生成されます（通常は `gemini-dispatch.yml` など）。このファイルには、IssueやPRのイベントをトリガーとして、`GEMINI_API_KEY` を使ってエージェントを実行する設定が含まれています。

### 3.2 ワークフローの確認とコミット

1.  生成されたファイル（`.github/workflows/*.yml` と `GEMINI.md`）を確認します。
2.  これらのファイルをリポジトリにコミットし、GitHubにプッシュします。

    ```bash
    git add .github/workflows GEMINI.md
    git commit -m "feat: Setup Gemini CLI GitHub Actions and context rules"
    git push
    ```

## 構築後の開発サイクル（テスト）

環境構築が完了したら、以下のサイクルで動作確認を行います。

1.  **Issue作成**: GitHubで新しいIssueを作成し、**ステップ2**で定義した`GEMINI.md`のルールに従って内容を詳細に記述します。
2.  **トリガー**: Issueに **`ready-for-ai`** ラベルを付与します。
3.  **AIの自律処理**: GitHub Actionsが起動し、AIエージェントがコードを修正します。
4.  **PRの確認**: エージェントが作成したPull Requestを確認し、コードとコミットメッセージが`GEMINI.md`のルールに従っているかをレビューします。

これで、ローカルPCでの作業を「AIが処理できない複雑な設計」や「最終的な承認」に限定する体制が整いました。
